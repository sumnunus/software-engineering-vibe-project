# software-engineering-vibe-project

소프트웨어공학 - 프로세스에 입각한 바이브코딩 프로젝트

## 프로젝트 개요

이 프로젝트는 소프트웨어공학 프로세스를 적용하여 바이브코딩 방식으로 소프트웨어를 기획, 설계, 구현, 테스트하고 그 효과와 한계를 분석하기 위한 과제이다.

개인 프로젝트로 진행하며, 만들 소프트웨어는 **외출 준비 통합 도우미 앱**이다. 아침 외출 준비 과정에서 투두, 날씨, 옷차림, 버스 도착 정보, 출발까지 남은 시간 등이 여러 앱에 흩어져 있어 확인이 번거로운 문제를 해결하는 것을 목표로 한다.

## 문제 인식

아침에는 제한된 시간 안에 준비물 확인, 복장 선택, 교통 상황 확인, 출발 시각 관리가 동시에 이루어진다. 현재는 사용자가 여러 앱을 오가며 정보를 확인해야 하므로 다음과 같은 불편이 발생한다.

- 외출 전 챙겨야 할 준비물을 놓치기 쉽다.
- 날씨와 기온을 확인한 뒤 적절한 옷차림을 다시 판단해야 한다.
- 자주 타는 버스의 도착 정보를 별도로 확인해야 한다.
- 설정한 외출 시각까지 남은 시간을 계속 의식해야 한다.

따라서 이 프로젝트는 외출 준비에 필요한 정보를 한 화면에 모아 보여주는 간단하고 실용적인 앱을 구현한다.

## 목표 기능

초기 목표 기능은 다음과 같다.

- 외출 전 준비물 투두 리스트
- 오늘 날씨와 현재 기온 표시
- 기온에 따른 옷차림 추천
- 사용자가 사전에 등록한 버스의 도착까지 남은 시간 표시
- 사용자가 설정한 외출 시각과 남은 시간 표시
- 아침 준비 상황을 빠르게 확인할 수 있는 단일 대시보드 화면

구현 범위는 요구사항 분석 및 설계 과정에서 조정될 수 있다.

## 적용할 소프트웨어공학 프로세스

본 프로젝트는 다음 흐름을 기준으로 진행한다.

1. 요구사항 분석
   - 사용자 문제 정의
   - 핵심 기능과 우선순위 도출
   - 기능 요구사항 및 비기능 요구사항 정리

2. 설계
   - 화면 구조 설계
   - 데이터 구조 설계
   - 주요 컴포넌트와 책임 분리

3. 구현
   - 바이브코딩을 활용한 기능 구현
   - 구현 과정에서 생성된 코드와 의사결정 기록
   - GitHub issue, commit, 문서 변경 이력을 통한 형상 관리

4. 테스트 및 품질 관리
   - 기능별 테스트 케이스 작성
   - 수동 테스트 및 필요 시 자동 테스트 수행
   - 결함 발견, 수정, 재검증 기록

5. 유지보수 및 회고
   - 구현 후 개선 사항 정리
   - 프로세스 적용의 장단점 분석
   - 바이브코딩의 효과와 한계 정리

## 바이브코딩 활용 계획

바이브코딩 도구는 프로젝트 진행 중 자율적으로 선정한다. 인공지능은 다음 활동에 활용한다.

- 요구사항 정리 및 누락된 요구사항 점검
- 화면 구성 아이디어 도출
- 코드 초안 생성 및 리팩터링 지원
- 테스트 케이스 작성 보조
- 문서화 및 회고 정리 보조

단, 최종 의사결정은 프로젝트 수행자가 내리며, AI가 제안한 결과는 검토 후 반영한다.

## 산출물 계획

- `README.md`: 프로젝트 개요와 진행 방식
- `docs/requirements.md`: 요구사항 분석 문서
- `docs/design.md`: 설계 문서
- `docs/test_report.md`: 테스트 및 품질 관리 보고서
- `docs/lessons_learned.md`: 프로세스 적용 교훈과 바이브코딩 효과 분석
- `docs/github_issues.md`: GitHub issue 등록 계획 및 초안
- `meeting_logs/`: 주기적 토의 및 진행 기록
- `screenshots/`: 완성된 소프트웨어 화면 캡처
- `src/`: 소프트웨어 구현 코드
- `tests/`: 테스트 코드 또는 테스트 자료

## 진행 기록 및 증빙 방식

과제 요구사항에 따라 프로젝트 착수 시점부터 지속적인 토의와 진행 기록을 남긴다.

- GitHub repository: <https://github.com/sumnunus/software-engineering-vibe-project>
- GitHub issue를 사용하여 요구사항, 설계, 구현, 테스트 작업을 관리한다.
- 주요 의사결정은 `meeting_logs/`에 날짜별로 기록한다.
- commit 이력을 통해 작업이 제출 직전에 몰아서 진행되지 않았음을 증빙한다.
- 기능 구현 후 화면 캡처를 `screenshots/`에 저장한다.
