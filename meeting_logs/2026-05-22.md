## 논의 배경

소프트웨어공학 과제에서는 단순히 소프트웨어를 완성하는 것뿐 아니라, 요구사항 분석에서 품질 관리에 이르는 개발 프로세스를 적용하고 그 과정에서 얻은 교훈을 정리해야 한다. 또한 과제 착수 시점부터 지속적으로 토의가 이루어졌음을 GitHub와 회의록 등으로 증빙해야 한다.

이에 따라 프로젝트 초기에 README, 회의록, 요구사항 문서, 설계 문서, 테스트 보고서, 회고 문서를 체계적으로 관리하기로 했다.

## 프로젝트 주제 논의

개인 프로젝트로 진행하며, 주제는 **외출 준비 통합 도우미 앱**으로 정했다.

이 앱은 아침 외출 준비 과정에서 여러 앱에 흩어진 정보를 한 화면에서 확인할 수 있도록 돕는 것을 목표로 한다. 사용자는 보통 아침에 다음 정보를 따로 확인해야 한다.

- 오늘 챙겨야 할 준비물
- 현재 날씨와 기온
- 기온에 맞는 옷차림
- 자주 타는 버스 도착 정보
- 설정한 외출 시각까지 남은 시간

여러 앱을 번갈아 확인하는 과정은 바쁜 아침 시간에 불편하고, 준비물을 빠뜨리거나 출발 시간을 놓치는 원인이 될 수 있다. 따라서 외출 준비와 관련된 정보를 하나의 대시보드에서 보여주는 앱을 기획하기로 했다.

## 초기 목표 기능

오늘 논의한 초기 기능 범위는 다음과 같다.

1. 외출 전 준비물 투두 리스트
2. 오늘 날씨 및 기온 표시
3. 기온별 옷차림 추천
4. 사전에 등록한 버스의 도착 예정 시간 표시
5. 사용자가 설정한 외출 시각과 남은 시간 표시
6. 위 정보를 한 화면에서 확인할 수 있는 대시보드

구현 난이도와 API 사용 가능성에 따라 실제 구현 범위는 요구사항 분석 단계에서 조정하기로 했다. 특히 버스 도착 정보는 실제 공공 API 연동이 어려울 경우, 프로토타입 단계에서는 사용자가 설정한 예시 데이터 또는 mock 데이터를 사용할 수 있다.

## 프로세스 적용 방향

프로젝트는 다음 단계로 진행하기로 했다.

1. 요구사항 분석
   - 사용자 문제와 사용 시나리오 정리
   - 기능 요구사항과 비기능 요구사항 분리
   - 우선순위 설정

2. 설계
   - 화면 구성 및 주요 컴포넌트 설계
   - 데이터 구조 설계
   - 외부 데이터 연동 여부 결정

3. 구현
   - 바이브코딩을 활용하여 기능별로 점진 구현
   - GitHub issue와 commit으로 작업 단위 기록

4. 테스트 및 품질 관리
   - 기능별 테스트 항목 작성
   - 정상 동작, 예외 상황, 화면 사용성 점검
   - 결함 수정 이력 기록

5. 회고
   - 프로세스 적용의 효과 정리
   - 바이브코딩이 요구분석, 설계, 구현, 테스트에 미친 영향 분석
   - 한계와 개선점 정리
