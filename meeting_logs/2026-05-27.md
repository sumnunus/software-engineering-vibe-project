## 진행 배경

이전 단계에서는 ReadyGo 대시보드 UI를 1차 구현하고, 화면 검토를 통해 카드 배치와 색상 등을 조정했다. 오늘은 Issue #4에 해당하는 준비물 투두 리스트 기능을 구현한다.

## 구현 전 확인한 의도

투두 기능 구현 전 프로젝트 수행자의 의도를 다음과 같이 확인했다.

- 기본 준비물 항목은 `지갑`, `이어폰`, `우산`, `노트북 충전기`로 설정한다.
- 추가 버튼을 누르면 카드 안에 입력창이 나타나도록 한다.
- 수정 아이콘을 누르면 해당 항목이 인라인 입력창으로 바뀌도록 한다.
- 삭제 시 확인 문구는 `삭제하기`로 한다.
- 빈 문자열 입력은 저장하지 않고 `내용을 입력해 주세요.` 메시지를 표시한다.
- 모든 항목이 완료되면 진행바 아래, 투두 목록 위에 작은 완료 배너를 표시한다.
- 새로고침 후에도 투두 상태가 유지되도록 localStorage에 저장한다.

## 소프트웨어공학 원칙 적용 방향

투두 기능은 화면 컴포넌트 안에 모든 로직을 몰아넣지 않고, 책임별로 나누어 구현하기로 했다.

- `storageService`: localStorage 접근을 캡슐화한다.
- `useTodos`: 투두 추가, 수정, 삭제, 체크, 체크해제 상태 변경을 응집도 있게 관리한다.
- `TodoPanel`: 화면 표시와 이벤트 연결을 담당한다.

이 구조를 통해 저장 방식이 바뀌더라도 UI 컴포넌트의 변경을 최소화하고, 투두 상태 변경 로직을 한 곳에서 관리할 수 있도록 한다.

## 중간 검토 및 수정 사항

브라우저에서 투두 기능을 직접 확인하며 다음 수정 의견을 반영했다.

- 삭제 확인을 브라우저 기본 확인창에서 앱 내부 확인 팝오버로 변경했다.
- 삭제 확인 팝오버에 `삭제할까요?`, `취소`, `삭제하기` 버튼을 배치했다.
- 빈 공간을 누르면 삭제 확인 팝오버가 닫히도록 했다.
- 삭제 확인 팝오버가 열린 상태에서 `Enter`를 누르면 삭제가 실행되도록 했다.
- `Escape`를 누르면 삭제 확인이 취소되도록 했다.
- 연속으로 준비물을 추가할 수 있도록, 추가 후 입력창이 계속 유지되게 했다.
- 빈 입력 오류가 표시된 상태에서 빈 공간을 누르면 추가 입력 상태가 취소되도록 했다.
