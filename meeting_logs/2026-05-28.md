## 구현 전 확인한 의도

기능 구현 전 프로젝트 수행자의 의도를 다음과 같이 확인했다.

- 날씨 위치는 브라우저의 현재 위치를 우선 사용한다.
- 현재 위치 권한을 사용할 수 없거나 API 호출이 실패하면 홍대입구 기준 임시 데이터를 보여준다.
- 날씨 정보는 현재 기온, 최저/최고 기온, 체감 온도, 강수량, 업데이트 시간을 포함한다.
- 날씨 카드에는 텍스트와 함께 날씨 아이콘을 표시한다.
- 옷차림 추천은 프로젝트 수행자가 제시한 기온별 옷차림 참고 이미지를 기준으로 정리한다.
- 사용자가 이해하기 쉬운 친근한 문장으로 추천 메시지를 표시한다.
- 기능 구현 후 화면 캡처를 남기고 README를 업데이트한다.

## 소프트웨어공학 원칙 적용 방향

날씨 기능은 외부 API, 위치 조회, 임시 데이터, 추천 규칙이 섞이기 쉬우므로 책임을 분리하여 구현하기로 했다.

- `locationService`: 현재 위치 조회와 fallback 위치 정책을 캡슐화한다.
- `weatherService`: Open-Meteo API 요청, 응답 정규화, 실패 처리 책임을 담당한다.
- `mockWeatherService`: API 실패 시 사용할 임시 날씨 데이터를 제공한다.
- `outfitRules`: 기온별 옷차림 추천 규칙을 UI와 분리한다.
- `WeatherPanel`: 날씨 데이터 표시, 로딩 상태, 추천 결과 출력만 담당한다.

이 구조를 통해 API 제공자가 변경되거나 옷차림 추천 기준을 조정하더라도 화면 컴포넌트 변경을 최소화할 수 있다. 또한 각 모듈의 역할을 좁게 유지하여 응집도를 높이고 결합도를 낮추는 방향으로 구현한다.
