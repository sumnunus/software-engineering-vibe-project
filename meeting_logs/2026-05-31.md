## 진행 배경

이전 단계에서는 관심 정류장 검색, 버스 도착 정보 표시, 30초 자동 갱신, mock fallback을 구현했다. 서울 버스 API 인증키는 활용신청 승인 후에도 API 서버에서 즉시 인식되지 않아 별도 후속 issue로 재검증하기로 했다. 오늘은 Issue #7에 해당하는 외출 시각 및 남은 시간 표시 기능을 구현한다.

## 구현 전 확인한 의도

- 상단 카드의 `수정` 버튼을 누르면 카드 안에서 바로 외출 시각을 입력한다.
- 남은 시간은 `1시간 12분 남았어요`, `23분 남았어요`, `지금 나가야 해요`처럼 표시한다.
- 남은 시간이 10분 이하이면 상단 카드 배경과 안내 문구를 연한 경고색으로 강조한다.
- 기존 개나리색 준비 게이지는 유지한다.
- 준비 게이지는 외출 1시간 전부터 차오르고, 외출 시각에 100%가 된다.
- 설정한 외출 시각은 localStorage에 저장하여 새로고침 후에도 유지한다.

## 소프트웨어공학 원칙 적용 방향

시간 계산, 저장, UI 상태를 분리하여 각 모듈이 하나의 책임에 집중하도록 한다.

- `timeUtils`: 현재 시각과 외출 시각의 차이, 표시 문구, 상태, 게이지 비율을 계산한다.
- `storageService`: 외출 시각의 localStorage 저장 및 복원을 캡슐화한다.
- `DepartureTimePanel`: 외출 시각 수정 폼, 1초 단위 갱신, 화면 표시를 담당한다.

이를 통해 시간 표시 기준이 바뀌더라도 계산 유틸리티를 중심으로 수정할 수 있고, 저장 방식이 바뀌어도 화면 컴포넌트의 변경을 줄일 수 있다.

- `npm run build` 명령을 실행하여 production build 성공을 확인했다.

- 고정된 현재 시각을 기준으로 대기, 활성, 긴급, 경과 상태를 각각 계산하여 문구와 게이지 비율을 확인했다.

- 프로젝트 수행자가 브라우저에서 수정 UI와 게이지 색상을 최종 확인했다.
